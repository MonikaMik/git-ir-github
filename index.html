Git

Git'as yra populiariausia kodo versijavimo programa.

Git komandos

  • init - sukuria git repozitorija dabartiniame aplankale
  • status - nurodo informacija apie lokalios git repozitorijos statusa
  • add - prideda nurodytus files prie commitui paruostu failu (stage'ina)
  • rm --cached - isima nurodytus failus is stage'inimo zonos
  • commit -m 'zinute' - sukuria kodo versija is stage'intu failu ir prideda zinute apie pakitimus (ne daugiau nei 50 simboliu)
  • restore (git checkout -- file) - grazina nurodytus failus i buvusio commit busena (panaikina naujus pakitimus padarytus po paskutinio commit)
  • push - is lokalios repozitorijos nusiuncia commit'us i isorine repozitorija
  • pull - is isorines repozitorijos atsiuncia commit'us i lokalia repozitorija
  • clone - nukopijuoja nurodyta repozitorija is isorines repozitorijos
  • checkout commitID|branchName - pereina i nurodyta kodo versija arba i kita branch
  • branch naujosSakosPavadinimas - sukuria nauja saka

Branch protection rules

Vienas svarbiausiu saku apsaugos taisykliu yra "Pull Request".(bandymas vienos sakos commitus sukelti i kita saka)
Pull Request dar galima suteikti "Require approvals", kuris reikalauja, kad nurodytas kiekis kitu projekto dalyviu patvirtintu Pull Requesta.
